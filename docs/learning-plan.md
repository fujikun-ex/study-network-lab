# ネットワーク学習計画

## 目標

OSI参照モデル L1〜L7 の各レイヤーを、具体的なイメージとともに理解する。
実機を購入して実際に操作しながら「これがこう動いている」を体感ベースで学ぶ。

---

## やること

- 実機（スイッチ・ルーター等）を購入して手を動かす
- 各レイヤーの役割と動作を実機検証で確認する
- 検証した内容を Markdown で記録・整理する
- 構成図・設定例・検証ログを再現可能な形で残す

### 予算

実機購入に使える予算: **〜10万円程度**

---

## L1〜L7 各レイヤー概要

| レイヤー | 名称                 | 主な役割                             | 代表的なプロトコル・機器          |
|--------|----------------------|--------------------------------------|----------------------------------|
| L1     | 物理層               | 電気信号・ビットの伝送               | ケーブル、ハブ、NIC              |
| L2     | データリンク層       | 同一ネットワーク内のフレーム転送     | Ethernet、VLAN、スイッチ         |
| L3     | ネットワーク層       | 異なるネットワーク間のパケット転送   | IP、ICMP、ルーター               |
| L4     | トランスポート層     | エンドツーエンドの通信制御           | TCP、UDP                         |
| L5     | セッション層         | 通信セッションの確立・維持・終了     | NetBIOS、RPC                     |
| L6     | プレゼンテーション層 | データ形式の変換・暗号化・圧縮       | TLS/SSL、JPEG、UTF-8             |
| L7     | アプリケーション層   | ユーザーが直接使うアプリの通信       | HTTP、DNS、SMTP、FTP             |

---

## 学習トピックと検証テーマ

### L1: 物理層を理解する

- **目標**: ケーブル種別・通信速度・信号伝達の仕組みを理解する
- **やること**
  - ケーブル（Cat5e / Cat6 など）の違いを調べる
  - NIC のリンク速度・全二重/半二重を確認する
- **わかること**
  - ケーブルの種類で通信速度や距離の限界が変わることがわかる
  - リンクアップ／ダウンのログがどのレイヤーの話かわかる

---

### L2: スイッチングとVLANを理解する

- **目標**: MACアドレス学習・フレーム転送・VLANの分離を理解する
- **やること**
  - スイッチのMACアドレステーブルを確認する
  - VLANを設定してネットワーク分離を検証する
  - タグVLAN（トランク）とアクセスポートの違いを試す
- **わかること**
  - スイッチがどのようにフレームを転送するかがわかる
  - 同じスイッチに繋いでもVLAN違いでは通信できないことが体感できる

---

### L3: ルーティングを理解する

- **目標**: IPアドレス・サブネット・ルーティングの仕組みを理解する
- **やること**
  - 静的ルートを設定して異なるサブネット間で通信させる
  - `ping` / `traceroute` でパケットの経路を確認する
  - DHCPサーバーを構築してIPを自動配布する
- **わかること**
  - デフォルトゲートウェイがなぜ必要かがわかる
  - ルーティングテーブルの見方と動作がわかる

---

### L4: TCP/UDPを理解する

- **目標**: コネクション確立・信頼性制御・ポート番号の役割を理解する
- **やること**
  - `tcpdump` / Wireshark で3ウェイハンドシェイクをキャプチャする
  - TCP と UDP の通信の違いを観察する
- **わかること**
  - TCPの再送制御・輻輳制御がなぜ必要かがわかる
  - ポート番号がどのようにアプリを特定するかがわかる

---

### L5〜L7: セッション・プレゼンテーション・アプリケーション層を理解する

- **目標**: HTTPSの仕組み・DNSの名前解決・TLSの暗号化を理解する
- **やること**
  - ローカルでDNSサーバーを立てて名前解決を確認する
  - HTTPと HTTPSの通信をキャプチャして違いを観察する
  - TLS ハンドシェイクの流れを追う
- **わかること**
  - ブラウザでURLを入力してからページが表示されるまでの流れが全レイヤーを通してわかる
  - 暗号化されていない通信がどれだけ危険かが実感できる

## 購入予定機器（候補）

| 機器                | 用途                         | 備考                  |
|---------------------|------------------------------|-----------------------|
| L2スイッチ          | VLAN・MACアドレス学習の検証  | Cisco Catalyst 系など |
| L3スイッチ or ルーター | ルーティング・NATの検証    | 中古でもOK            |
| Raspberry Pi × 2    | サーバー・クライアント役     | DHCPサーバー構築など  |
| LANケーブル各種     | 物理層の確認                 | Cat5e / Cat6          |
